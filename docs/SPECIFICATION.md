# クルーズファインダー仕様書

最終更新: 2026-07-02

## 目的

クルーズ旅行会社のサイトは、会社ごとに検索性、表記、価格表示、詳細ページへの導線がばらつく。そのため、このツールでは複数サイトの掲載情報を横断取得し、利用者が出発日、船、料金、方面、販売会社で比較できる静的サイトとして提供する。

このツールは予約システムではない。比較しやすい一覧と、各販売会社の詳細・予約ページへの導線を提供する。

## 公開形態

現在はGitHub Pagesで公開できる単体HTMLとして運用する。

- 公開ファイル: `index.html`
- 正データ: `cruises.json`
- 表計算確認用: `cruises.csv`
- アフィリエイト差し替え元: `affiliate_links.csv`

`index.html` には `cruises.json` の内容を `DATA` として埋め込む。これにより、サーバーサイドAPIなしで検索、絞り込み、一覧表示が動く。

## 将来のVPS方針

将来さくらVPSへ移行しても、基本設計は `静的配信 + 定期更新ジョブ` のままにする。

- Nginxなどで `index.html` を静的配信する。
- ユーザーアクセス時にスクレイピングや外部サイト取得を走らせない。
- cronまたはsystemd timerで `npm run refresh` を実行する。
- 更新ジョブが `cruises.json`、`cruises.csv`、`index.html`、`affiliate_links.csv` を更新する。
- 将来DBを使う場合も、まずは `cruises.json` を生成成果物として残す。

想定配置:

```text
/var/www/cruise-finder/current
```

cron例:

```cron
15 4 1 * * cd /var/www/cruise-finder/current && /usr/bin/npm run refresh >> /var/log/cruise-finder-refresh.log 2>&1
```

## データ取得元

### ベストワンクルーズ

スクリプト: `import_best1_cruises.mjs`

ベストワンクルーズの一覧APIから取得する。ページングして取得し、価格、日付、泊数、船名、船会社、方面、出発港、寄港地を正規化する。

重要な注意:

- `comment3` の項目順は商品によって異なる。
- 泊数は固定位置ではなく、`\d+泊\d+日` に一致する項目から探す。
- MSCなど一部商品は項目順が違うため、固定位置だけを見ると落ちる。
- 価格が0または取得不能な商品は除外する。

### BUTE

スクリプト: `import_partner_cruises.mjs`

BUTEは通常のHTML取得が可能なため、船別ページや掲載ページから詳細URLを集め、詳細ページを巡回して取得する。

重要な注意:

- ページ全体にはナビゲーションや別船の文言も含まれるため、船会社判定はページ全文ではなくタイトル中心に行う。
- 料金が明確に円建てで取得できるものだけ採用する。
- 価格不明、問い合わせ前提、0円相当は除外する。

### クルーズプラネット

スクリプト: `import_partner_cruises.mjs`

クルーズプラネットはローカルやGitHub Actionsからの直接fetchが403になるページがある。そのため、現時点では確認済み詳細URLを種データとして登録する。

今後の拡張案:

- VPS上で取得可否を再検証する。
- 取得できるURLだけを巡回対象にする。
- 403が続く場合は、手動確認済み詳細URLリストを増やす運用にする。

## 更新コマンド

通常更新は以下で行う。

```sh
npm run refresh
```

内部では以下を順に実行する。

```sh
npm run import:best1
npm run import:partners
npm run update
npm run affiliate
```

各コマンド:

- `import:best1`: ベストワンクルーズを取得する。
- `import:partners`: BUTEとクルーズプラネット由来のデータを取得、反映する。
- `update`: 掲載URLの到達確認、最終確認日、HTTPステータスなどを更新する。
- `affiliate`: `affiliate_links.csv` の `affiliateUrl` を `cruises.json` と `index.html` に反映する。

## 重複除外ルール

同じクルーズが複数サイトに掲載されるため、重複は除外する。

現在の重複キー:

```text
正規化した船名 + 出発日
```

同じキーの行が複数ある場合は、`priceFrom` が最も安い行だけを残す。

正規化では以下を行う。

- 空白を除去する。
- 中黒などの表記ゆれを軽く吸収する。
- 末尾の `号` を除去する。
- 小文字化する。

注意:

同じ船名、同じ出発日でも、発着港違い、区間乗船、航空券付き、ホテル付き、添乗員付きなど別商品である可能性がある。現時点では一覧性と価格比較を優先し、船名と出発日が同じなら同一クルーズとして扱う。

将来必要なら、重複キーを以下に拡張する。

```text
船名 + 出発日 + 泊数 + 出発港 + 到着港
```

## 価格ルール

- `priceFrom` は最安目安価格。
- 通貨は原則JPY。
- 0円、空欄、価格不明は掲載しない。
- USDなど外貨のみの掲載は、現時点では円換算しない。必要になったら通貨フィールドと換算日を明示して対応する。
- 料金、空席、客室条件は販売会社ページで最終確認する。

## クラブルーム級

船会社ごとに上位客室の呼び方が異なるため、UI上は `クラブルーム級` としてまとめる。

例:

- MSC: MSCヨットクラブ
- NCL: The Haven級
- ロイヤル・カリビアン: Royal Suite Class級
- プリンセス: スイート/リザーブ・コレクション級
- キュナード: プリンセス/クイーンズ・グリル級
- 飛鳥、三井、にっぽん丸、ラグジュアリー船: 上位スイート級

これは厳密な客室保証ではなく、絞り込み用の候補分類である。対象客室と特典は販売会社ページで確認する。

## 子連れスコア

子連れスコアは目安であり、実際の年齢制限、乳幼児対応、ベビーベッド、託児、子供料金は販売会社や船会社ページで確認する。

大まかな考え方:

- `S`: ファミリー向け大型船、MSC、ロイヤル・カリビアン、コスタなど。
- `A`: 大型船、プリンセス、ノルウェージャンなど。
- `B`: 一般的なクルーズ。
- `C`: 子連れ確認がより必要なもの。

## アフィリエイト

`affiliate_links.csv` に `affiliateUrl` を入れると、画面の `詳細・予約へ` は `affiliateUrl` を優先する。空欄なら通常の `bookingUrl` に遷移する。

CSV形式:

```csv
source,title,bookingUrl,affiliateUrl,memo
```

方針:

- アフィリエイトURLは生成データに直接手書きしない。
- `affiliate_links.csv` に入力し、`npm run affiliate` で反映する。
- 更新処理でアフィリエイトURLが消えないよう、URLまたはタイトルで既存行を引き継ぐ。

## データ品質チェック

更新後に確認すること:

```sh
node -e 'const d=require("./cruises.json"); console.log({rows:d.length, zero:d.filter(r=>Number(r.priceFrom||0)<=0).length, placeholder:d.filter(r=>r.status==="要公式確認").length})'
```

確認観点:

- 0円データがない。
- `要公式確認` の仮データが残っていない。
- 主要船名で検索できる。
- 重複キーが過剰に残っていない。
- 画面でJavaScriptエラーが出ない。

ブラウザ確認の目安:

- 初期表示件数が0でない。
- `MSC` 検索でBest1由来のMSCが出る。
- `ベリッシマ` 検索でBest1、BUTE、クルーズプラネット由来の最安値採用結果が出る。
- 販売会社フィルタでBest1、BUTE、クルーズプラネットを選べる。

## 触るときの注意

- 取得元サイトのHTMLやAPIは変わる前提で作業する。
- 表記ゆれを完全に消そうとしすぎず、まず一覧性と価格比較を優先する。
- 予約可否や空席はこのツール内で断定しない。
- URLがトップページに飛ぶだけの場合は、できるだけ詳細ページや検索結果ページに差し替える。
- 大きなデータ更新では `cruises.json`、`cruises.csv`、`index.html` の差分が大きくなるのは正常。
- ロジック変更とデータ大量更新を同時に行った場合は、最終件数、0円件数、主要検索結果を必ず確認する。
